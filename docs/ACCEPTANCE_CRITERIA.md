# ACCEPTANCE_CRITERIA.md

## General Acceptance Criteria

The MVP is acceptable when:

1. FastAPI server runs locally.
2. `GET /api/health` works.
3. `GET /api/daily-report/latest` returns a complete mock daily report.
4. `POST /api/analyze-stock` returns a complete mock stock analysis.
5. Unit tests pass.
6. No prohibited investment instruction language appears in responses.
7. Every score includes raw data, source, benchmark, trend, confidence, limitations, and missing data.
8. Missing data is explicit.
9. `not_investment_advice` is always true.

## Prohibited Language Test

Automated tests should fail if API responses contain:

- buy
- sell
- hold
- liquidate
- exit all
- sell half
- must invest
- guaranteed

## Engine Acceptance Criteria

### Macro Regime Engine

Must classify at least:

- normal
- fear_crisis
- inflation_pressure
- recession_warning
- overheated

### Market Timing Engine

Must calculate:

- Fed easing score
- index drawdown stabilization score
- Fear & Greed score
- VIX confirmation score
- company cash score
- revenue growth score
- founder CEO + insider buying score

### Overheat Engine

Must calculate:

- index overextension
- Fear & Greed greed score
- media hype
- YouTube hype
- user-reported social heat

### Leadership Engine

Must calculate all 20 criteria.

### Smart Money Engine

Must calculate:

- 13F institutional support
- Form 4 insider signal
- options abnormal activity

### Future Industry Radar

Must return at least 5 themes using mock data.

## UI Acceptance Criteria

When frontend is implemented, it must show:

- daily report dashboard
- stock analysis page
- score cards
- expandable raw data panels
- benchmark and trend displays
- limitations and missing data

## Phase 6 UI Acceptance Criteria

Phase 6 is complete when:

1. Frontend runs locally with React, TypeScript, and Vite.
2. Daily Report page loads `GET /api/daily-report/latest`.
3. Stock Research page submits a ticker to `POST /api/analyze-stock`.
4. Major report sections are visible: market state, timing, themes, candidates, smart money, risk reference, missing data, and limitations.
5. Raw data, benchmark, trend, confidence, limitations, and missing data are available through expandable panels.
6. Frontend includes a development warning for restricted instruction language in API payloads.
7. Backend tests pass.
8. Frontend build and frontend tests pass when Node/npm dependencies are installed.
9. README includes Windows PowerShell commands for backend and frontend.

## Phase 7 Validation Acceptance Criteria

Phase 7 is complete when:

1. Backend tests pass.
2. Frontend build passes.
3. Frontend tests pass when dependencies are installed.
4. Daily report endpoint returns schema-compliant mock data.
5. Analyze-stock endpoint returns schema-compliant mock data.
6. Every score-like object includes raw data, derived metrics, benchmark, trend, source, source date, confidence, limitations, and missing data.
7. Backend forbidden-language utility detects direct instruction language while allowing research labels.
8. No forbidden instruction language appears in API response string values.
9. API schema documentation matches the current response structure.
10. README includes Windows VSCode run instructions and notes that live APIs are not connected yet.

## Phase 1 Acceptance Criteria

Phase 1 is complete when:

1. Mock daily report is generated by pipeline.
2. Daily report API returns valid schema.
3. Analyze-stock API returns valid schema.
4. Tests cover normal, fearful, overheated, missing data scenarios.
